## 严重BUG报告与解决方案

### 自动保存和备份失效

<Callout>
问题描述：

1.5+版本，把json文件的打开方式设置为此软件，直接双击打开json文件，打开文件时自动保存和备份失效。

</Callout>
<Callout>
解决方法：

不要把json文件的打开方式设置为此软件，实际上也不应该这样设置，因为json文件是一种通用的文件，不应该被设置其他软件的打开方式。应该直接打开此软件本体，（使用快捷方式或者直接打开exe文件）

打开后按 `shift+#` 打开最近文件列表，然后再切换文件。

——2025年4月12日
</Callout>
### 舞台上全部连线丢失

<Callout>
原因是把工程文件添加到了启动文件中，或者把json文件的打开方式设置为此软件，直接双击打开了json文件。

可能启动文件体积较大，加载时连线加载失败导致全部连线丢失，建议仅在启动文件中记录简单内容。

大型笔记文件可以添加到启动列表以供快速手动切换，但不建议标记图钉。📌

——2025年4月9日
</Callout>
